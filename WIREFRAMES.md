# ワイヤーフレーム（モバイルファースト）

目的
- モバイル優先でページ構成と主要コンポーネントの配置・挙動を定義する。

共通ルール
- ベース間隔: 8px スケール（8,16,24,32...）
- コンテナ横パディング: 16px (mobile) / 24px (tablet+) / 32px (desktop)
- テキストサイズ（目安）: body 16px, h1 34px (mobile)
- アニメーション: 署名はロードで描画、各要素は `FadeIn` で遅延表示

## 1. ヘッダー（ナビ）
- 表示 (mobile): ハンバーガー + ロゴ（左）
- 挙動: スクロールでバックドロップ（透過→ソリッド）

## 2. ヒーロー（最上部）
- 要素（縦順）:
  1. `Signature`（SVG手描きアニメ）
  2. サイトタイトル（`h1`）
  3. サブテキスト（職種など）
  4. CTA ボタン（プロフィール / 作品へ）
- レイアウト: 中央寄せ、上下パディング 48px
- アニメ: `Signature` 描画完了後、`h1` とサブテキストをフェードイン（0.2s スタガー）

## 3. Works セクション（プロジェクト一覧）
- モバイル: 単一カラムのカード列（カード間 margin 16px）
- カード内容: サムネイル 16:9、タイトル、タグ、短い説明
- タップ挙動: カードタップで詳細モーダル（または別ページ）
- アニメ: スクロールで `FadeIn`（staggerChildren）

## 4. About セクション
- 縦積みレイアウト: 見出し、プロフィールテキスト、スキルリスト（chip 表示）
- アニメ: テキストとスキルは順にフェードイン

## 5. Contact セクション
- シンプルなフォーム（名前、メール、メッセージ） + 送信ボタン
- フォーム送信時のフィードバック（成功/失敗）を明確に表示

## 6. フッター
- コピーライト + 小さなリンク（SNS）

## ブレイクポイントとデスクトップ調整
- Tablet (min-width: 768px): Hero の `h1` を大型化、Works を 2カラムに
- Desktop (min-width: 1024px): Works を 3カラム、Hero を左右レイアウト（左にテキスト、右に大きな署名SVG）

## 注記（アニメーション細則）
- `Signature` は画面読み込みで1回のみ実行（`once`）
- 主要な要素は `framer-motion` を利用して `opacity` と `y` のトランスフォームでフェード & スライド
- ホバーやフォーカスはトランジション 180-220ms、ボタンはスケール 1.02
